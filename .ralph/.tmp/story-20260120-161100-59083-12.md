### US-009: Implement trace errors and costs commands
Status: in_progress
Depends on: US-006

Description:
As a user, I want specialized commands for finding errors and understanding costs.

Acceptance Criteria:
- [ ] 'trace errors' finds traces with errors in observations
- [ ] 'trace costs' shows cost breakdown by model, trace, or time period
- [ ] Errors command shows: trace ID, error message, timestamp, observation that failed
- [ ] Costs command shows: total cost, cost per model, cost per trace (top N)
- [ ] Example: 'trace errors --since 24h' -> errors in last 24 hours
- [ ] Example: 'trace costs --group-by model' -> cost breakdown by model
- [ ] Negative case: No errors found -> 'No errors in the specified time range'
- [ ] ISC reference: rows 17, 18
